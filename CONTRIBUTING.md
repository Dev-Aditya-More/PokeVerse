# Contributing to Pokéverse

Thanks for your interest in contributing to Pokéverse.
All contributions—code, documentation, ideas, and bug reports—are welcome, provided they align with the project’s goals and standards.

## Ways to Contribute

You can contribute by:

- Reporting bugs

- Suggesting features or improvements

- Improving documentation

- Submitting code via pull requests

- Before contributing, please read this document carefully.

## Ground Rules

By participating, you agree to:

- Follow the Pokéverse Code of Conduct

- Be respectful and constructive in discussions

- Avoid low-effort, spammy, or unfocused contributions

- This project values clarity and quality over quantity.

## Reporting Bugs

When reporting a bug, please include:

- A clear and concise description

- Steps to reproduce the issue

- Expected vs actual behavior

- Device, OS version, and app version (if applicable)

- Screenshots or logs, if available

- Incomplete reports may be closed without action.

## Feature Requests

Feature requests should:

- Solve a real problem

- Be clearly scoped

- Explain why the feature is useful

- Vague ideas like “add more features” or “make it cooler” will not be considered.

## Development Setup

Before submitting code:

- Ensure the app builds and runs without errors

- Follow existing project structure and architecture

- Avoid introducing unnecessary dependencies

- Keep changes focused and minimal

- If unsure about an approach, open an issue before starting work.

## Pull Request Guidelines

All pull requests should:

- Address a single concern (bug fix or feature)

- Include a clear description of changes

- Reference related issues, if any

- Avoid unrelated formatting or refactoring

- Pull requests may be requested to change, improve, or close if they do not meet project standards.

## Code Style & Quality

- Follow existing naming conventions and formatting

- Write readable, maintainable code

- Prefer explicit logic over clever tricks

- Add comments where intent is not obvious

- Code that sacrifices clarity for brevity will be rejected.

## Reviews & Feedback

- All submissions are reviewed

- Feedback is given with the goal of improving the project

- Reviews are not personal—take them objectively

- Not all contributions will be accepted.

## License & Ownership

By contributing, you agree that:

- Your contributions may be modified or removed

- Your contributions are provided under the same license as the project

- You grant Pokéverse the right to use your contributions without restriction
